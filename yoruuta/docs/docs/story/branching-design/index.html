分岐設計書

目的​

本資料は、共通ルートから個別ルート、BAD END への分岐条件を定義する。 好感度だけでなく、主人公が誰の夜にどう関わったかを分岐条件に含める。

設計方針​

基本原則: 蓄積型ルート決定​

  • ルートはプレイヤーの明示的な選択ではなく、蓄積された好感度・信頼・関わりの結果として自然に決まる
  • 「誰を追うか」ではなく「どう向き合ってきたか」がルートを決める
  • プレイヤーは「気がついたらこの人のそばにいた」と感じるべきである
  • 分岐は恋愛選択であると同時に、関わり方の選択である
  • 正解選択の積み上げではなく、相手に合った支え方を選べるかを問う
  • BAD END は、相手を傷つける明確な失敗として設計する

第4章の選択: 態度の選択であり、人の選択ではない​

第4章の崩壊後、プレイヤーに提示される選択は「誰を追うか」ではなく「この状況にどう向き合うか」という態度の選択である。この選択は好感度を微調整し、project_intact に影響するが、ルートを直接決定しない。

管理する主要値​

共通値​

  • affinity_saya
  • affinity_chihiro
  • affinity_yuzuki
  • affinity_akari
  • trust_saya_step1
  • trust_chihiro_step1
  • trust_yuzuki_step1
  • trust_akari_step1
  • saw_core_a
  • saw_core_b
  • saw_core_c
  • saw_core_akari
  • project_intact

分岐用の意味​

  • affinity_*
    • 共通ルート全体の積み上げ。継続して相手に合った反応を選べたかを見る
  • trust_*_step1
    • 第3章で、その相手の痛みに対して踏み込み方を誤らなかったかを見る
  • saw_core_*
    • その相手の傷の核心を、主人公が一度でも目撃したか
  • project_intact
    • 第4章で主人公が企画を人より優先していないかを見る
  • affinity_akari / trust_akari_step1 / saw_core_akari
    • fallback 用の補助導線。メインヒロイン不成立時に、外側から企画を支える相手を見ていたかを測る

  • affinity_chihiro: 8
  • affinity_yuzuki: 8
  • affinity_akari: 3

分岐に使う閾値（二段階判定）​

分岐判定は以下の二段階で行う。

第一段階: ルート進入の最低条件​

  • affinity_saya >= 3
  • affinity_chihiro >= 3
  • affinity_yuzuki >= 3

この閾値を下回ると、trust や saw_core の状態に関わらずルート不成立になる。

第二段階: trust 不要のショートカット条件​

  • affinity_saya >= 6
  • affinity_chihiro >= 6
  • affinity_yuzuki >= 6

この閾値を超えていれば、trust_*_step1 の判定をスキップしてルートに進入できる。 共通ルートで十分に寄り添い続けたプレイヤーへの救済導線として機能する。

この二段階設計は、

  • 最低限の関わりを持たなければルートに入れない
  • ただし高い信頼があれば、特定の選択肢を逃しても到達できる
  • メイン導線を外しすぎると届かない

というバランスを狙っている。

ルート決定アルゴリズム​

概要​

第4章の崩壊後、以下の手順でルートを自動決定する。プレイヤーに「誰を選ぶか」は問わない。

判定フロー​

1. project_intact チェック
   → false なら route_fail（共通BAD）

2. 各ヒロインの資格判定
   以下のすべてを満たすヒロインを「資格あり」とする:
   a. affinity_X >= 3（最低条件）
   b. trust_X_step1 = true OR affinity_X >= 6（信頼ゲート）
   c. saw_core_X = true（核心の目撃）

3. 資格ありヒロインが0人 → Akari fallback 判定へ

4. `affinity_akari >= 2` かつ `trust_akari_step1 = true` かつ `saw_core_akari = true`
   → あかりEND

5. あかり条件を満たさない場合、
   全ヒロイン `affinity >= 3` かつ `saw_core` 全取得
   → 友情END

6. 上記も満たさない → route_fail（共通BAD）

7. 資格ありヒロインが1人 → そのヒロインのルートへ

8. 資格ありヒロインが複数 → 最も好感度が高いヒロインのルートへ

9. 好感度が同率 → trust_step1 を持つ方を優先
   （trustは「第3章で正しく踏み込めた」ことを意味し、より深い関わりの証）

10. それでも同率 → saw_core の取得順序で判定
   （先に核心を見た = より早く相手の内面に触れた）

11. 全ヒロインが高水準で資格あり → true_route_flag をセット
   （3ルートクリア後のTrue Route解放条件の一部として使用可能）

実装上の注意​

エンジンの @if は単一条件のみサポートするため、複合条件判定はラベルチェーンで実装する。 各ヒロインの資格チェックを段階的に行い、資格ありヒロインの好感度を比較してルートを決定する。

具体的な @if チェーンの実装は scene_04_route_split.md を参照。

共通ルートの主要選択​

第0章​

  • 好感度に影響する選択肢は置かない
  • 演出的選択（結果が合流する選択肢）は操作感の維持のために許容する
  • まずは作品の空気と、主人公の基本姿勢を線形で見せる
  • 分岐管理は第1章以降から始める

第1章​

  • Bの強い言葉に、言い返すか、受け流すか、論点だけ拾うか

  • 推奨は「感情で返さず、論点だけ拾う」

  • Bは対等さに敏感なので、哀れみは減点になる

  • Cの無気力な態度に、励ますか、放置するか、隣に座るか

  • 推奨は「励ましすぎず、同じ場所にいる」

第2章​

  • 企画の役割分担で誰に肩入れするか
  • 共通ルートの信頼加算を複数回入れる
  • ここでは露骨な恋愛選択ではなく、実務上の寄り添い方として見せる

第3章​

  • Aの未完成原稿に対して、感想を急ぐか、返答を保留するか
  • Bの怒りに対して、正論で押すか、怒りの原因を待つか
  • Cの諦めに対して、未来を語るか、現在の感覚を聞くか

この章から、各ヒロインの核心を見るイベントに到達可能にする。

第4章（態度の選択）​

崩壊後の選択肢は「誰を追うか」ではなく「状況にどう向き合うか」を問う。

  • 「企画書を取り戻しに行く」 → project_intact 維持 / 好感度微減のリスク
  • 「……少し、頭を冷やそう」 → 冷静さによる好感度微増 / project_intact = true（冷静に考えた結果、企画を手放さない再確認に至る）
  • 「今は、みんなの方が大事だ」 → 好感度微増 / project_intact を false にするリスク

これらの選択は好感度を ±1 程度調整し、project_intact に影響するが、ルートを直接決定しない。ルートは蓄積された好感度の総合値で自動決定される。

BAD END 設計​

A系 BAD​

原因​

  • 本人の準備ができていないのに発表を急がせる
  • 過去の失敗を勝手に美談化する
  • 企画成功のためにAの声を消費する

結果​

  • Aが完全に沈黙する
  • 主人公は「届けばいいわけではない」と知る

B系 BAD​

原因​

  • Bを弱い立場として扱いすぎる
  • 家庭問題へ無断で介入する
  • 怒りを落ち着かせることだけを優先する

結果​

  • Bは主人公を切り捨てる
  • 企画も関係も壊れ、怒りだけが残る

C系 BAD​

原因​

  • 「頑張れ」で押す
  • 希望を押しつける
  • 諦めを理解したふりだけして放置する

結果​

  • Cは静かにいなくなる
  • 主人公は見失ったことに後から気づく

共通BAD​

原因​

  • 3人の心より企画の体裁を優先する
  • 噂や外圧に流される
  • 何も決めず先延ばしにする

結果​

  • 企画は中止
  • 誰の夜にも触れられないまま終わる

隠しEND 条件​

共通設計方針​

  • 隠しENDは通常GOOD ENDの後日談として挿入される
  • 通常ENDをクリアした上で、高好感度 + 全信頼フラグ + 特定条件を満たしている場合にのみ到達可能
  • シーン冒頭の @if チェーンで条件不足の場合は通常ENDに戻す
  • 隠しENDは「ご褒美」ではなく「この関係の先に何があるか」を見せるエピローグ
  • 副次ヒロインの隠しENDは、通常ENDを上書きする別勝利条件にしない
  • 副次ヒロインの隠しENDは「そのルートで外側にいた支え手を見る」ための補助的エピローグとして扱う
  • したがって、本編ヒロインの感情決着が終わったあとにのみ開く
  • 泣きや感動の核は 人気サブキャラへのご褒美 ではなく、支えていた側の喪失 / 献身 / 置き去りにした時間の回収 に置く
  • 最後は大きな告白より、ようやくその人自身が見られた という静かな受け取りで締める

  • 文化祭から二週間後。咲夜が校内の中庭で初めて人前で歌う
  • 朔の名前を呼ぶ時の声が変わっている――ためらいが消え、自分の声として呼んでいる
  • 新しい歌は凪の言葉の書き直しではなく、咲夜自身の言葉で書かれたもの

シーンファイル​

物語構造: 逆転構造​

個別ルートでは常に「聞く側」だった朔が、True Routeで初めて「自分の夜を聞かれる側」に回る。

前提: 朔が各ルートで学ぶもの​

ルート学び態度
A（咲夜）背中を押さない。選ばせる引き受ける覚悟を待つ
B（千紘）全部を背負わない。対等に任せる自分の範囲を明言する
C（結月）何も決めさせず隣にいる沈黙に耐える

シーン構成​

  1. 起点（第4章分岐）: 三人のうち誰か一人を追おうとするが、三人が先に動いている。咲夜は自分で旧音楽室に戻り、千紘は企画の立て直しを始め、結月は静かに残る。朔は「追う相手がいない」状態に置かれる
  2. 朔の露出: 湊が「お前、結局自分のことは何も言わないよな」と核心を突く。朔の「夜」―― 聞くことに逃げていたこと、自分には声にする何かがないと感じていること ―― が初めて表に出る → saku_night_exposed = true
  3. 逆転: 三人が、個別ルートで朔から受け取ったものを朔に返す。咲夜は朔の沈黙を待ち、千紘は朔の範囲を問い、結月は黙って隣にいる
  4. 着地: 文化祭の本番で三人がそれぞれの形で声を出す。朔は舞台に立たないが、自分が声にすべきものがあったと知った上で聞く側に立つ → saku_night_resolved = true

個別ルートとの差異​

項目個別ルートTrue Route
朔の立場一人に深く踏み込む自分の夜を聞かれる側に回る
クライマックスヒロインが自分で選ぶ朔自身が「聞く側」を選び直す
着地の感情二人の関係の変化朔が自分の空虚と向き合った上で、聞くことを引き受ける
三人の立場朔に支えられる朔を支える側に回る（逆転）

全ルートクリアの必然性​

CLANNADは「善意の蓄積が奇跡を生む」、リトバスは「繰り返しプレイが虚構世界のループだった」。 本作では**「プレイヤーが各ルートで選んだ態度が、朔に返ってくる」**構造にすることで、プレイヤーの行為そのものに意味を与える。

演出上の注意​

  • BAD END でも大事件にしない
  • 関係が壊れた結果として静かに終わる方が、本作には合う
  • どの失敗も、主人公が善意だけでは足りないと知る形にする
  • ルート確定の瞬間をプレイヤーが意識しないこと: 「誰のルートに入ったか」ではなく「気がついたらこの人のそばにいた」という感覚を優先する

実装メモ​

  • 好感度は長期蓄積、trust_*_step1 は章固有の関わり方判定として分ける
  • 分岐前の選択肢は、恋愛色を濃くしすぎない
  • ルート確定は第4章終盤の自動判定で行う（プレイヤーの明示的な人物選択を挟まない）
  • @if チェーンによるラベル分岐で、資格判定 → 好感度比較 → タイブレークを段階的に処理する
  • ルート内では、さらに正規エンドと BAD END を分ける選択肢を置く
  • サブストーリーの好感度効果は @affinity コマンドで処理する
