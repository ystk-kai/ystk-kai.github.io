チャプター区切りルール

目的​

チャプターの分割基準を明確にし、以下の問題を防ぐ。

  • タイムラインの逆行・矛盾
  • チャプター間のボリューム不均衡
  • 命名規則の不統一
  • 「なぜここで章が切れるのか」の根拠不在

現状の課題と推奨方針​

課題: 章番号とプレイ順の乖離​

chapter-outline.md では「章番号は設計上の分類であり、プレイ順とは異なる」と記されているが、実際のシナリオファイルのタイムライン（index.md の day_range）と設計文書のタイムラインにも乖離がある。

設計文書のプレイ順:
  Ch00(Day1) → Ch01(Day2-5) → Ch02(Day5-10) → Ch05(Day10-20)
  → Ch03(Day15-22) → Ch06(Day20-28) → Ch07(Day28-32) → Ch04(Day32-35)

実シナリオの day_range:
  Ch00(Day1-2) → Ch01(Day2-4) → Ch02(Day3-10) → Ch03(Day9-10)
  → Ch04(Day11-12) → Ch05(Day5-10) → Ch06(Day11-14) → Ch07(Day19-22)
  → Ch08(Day23-24) → Ch09(Day25-26) → Ch10(Day26-27)

設計文書と実装の両方でタイムラインが異なり、Ch05 の逆行問題が発生している。また Ch08〜Ch10 は設計文書に記載がない。

推奨方針: 章番号＝プレイ順に統一する​

章番号とプレイ順を一致させる。 ADV において、プレイヤーは「第X章」を順序として認識する。設計上の分類と実行順序を分離するメリットより、混乱のコストの方が大きい。

うたシリーズを含む大半の ADV 作品では章番号はそのままプレイ順であり、本作もこれに従う。設計上のグルーピング（企画フェーズ、危機フェーズ等）はドキュメント側のタグや注釈で管理し、章番号自体には持たせない。

チャプター区切りの5原則​

原則1: 感情の弧（アーク）で区切る​

チャプターの境界は 感情的な転換点 に置く。日付の変化や場面転換ではなく、「このチャプターで何が変わったか」を一文で言えること。

○ 「日常の蓄積により、この時間がかけがえないものだと感じるまで」
○ 「企画の内容が外に漏れ、各人の傷が表面化するまで」
× 「Day 9-10 のイベント」（日付ベース）
× 「放課後に起きたこと全部」（時間帯ベース）

チャプターの感情弧を一文で定義し、index.md に emotional_arc として記載する。

<!-- emotional_arc: 日常の積み重ねにより「この時間が消えたら」の恐怖の種を蒔く -->

原則2: タイムラインは前進のみ​

チャプター間でタイムラインを逆行させない。day_range は前のチャプターの終了日以降から開始すること。

○ Ch02(Day5-10) → Ch03(Day10-20) → Ch04(Day15-22)
   隣接チャプター間の重複は許容（並行する出来事）

× Ch02(Day3-10) → Ch03(Day9-10) → Ch04(Day11-12) → Ch05(Day5-10)
   Ch05 が Ch04 の後に Day 5 へ戻っている

回想シーンが必要な場合は、シーン内のフラッシュバックとして処理する。チャプター自体の day_range は前進を維持する。

原則3: ボリュームの目安を守る​

区分行数目安シーン数目安備考
プロローグ300〜800行3〜5世界観定義に集中。長すぎると導入が重い
通常チャプター1,200〜2,500行6〜12読者が1セッションで読み切れる単位
クライマックス章2,000〜3,000行8〜14感情のピークは密度を上げてよい
エピローグ/ボーナス300〜600行2〜4余韻に徹する。新情報は入れない

目安を ±30% 以上外れるチャプターは、分割または統合を検討する。

原則4: チャプター番号体系​

共通ルートは chapter_00 から連番。個別ルート以降は帯域を分ける。

帯域用途例
00〜10共通ルートchapter_00_night_entry
11〜19メインヒロインルートchapter_11a_saya_silence
20True Routechapter_20_true_route
21〜29合流系エンディングchapter_22_friendship_end
31〜39サブヒロインルートchapter_31_kotoha_route
91〜99ボーナスシナリオchapter_91_minato_bonus

メインヒロインルートの章分割は a/b/c サフィックスを使う。

サフィックス役割感情の位置
a傷の表面化痛みの認識
b対峙と失敗最も苦しい場所
c選択と着地カタルシスまたは喪失

原則5: タイトル形式の統一​

共通ルート:     第X章 - サブタイトル
ヒロインルート: キャラ名ルート 第X章 - サブタイトル
合流/エンド:    サブタイトルのみ
ボーナス:       ボーナスEND - サブタイトル

プロローグは「プロローグ - サブタイトル」とする（「第0章」は使わない）。

Ch07 統合の検討​

Ch07（前夜祭の灯）は 802行と短い。Ch06（嵐の前の凪）と統合する案もあるが、感情の弧が異なる（凪の静寂 vs 前夜祭の高揚）ため、独立チャプターとして維持する。今後のシーン追加で 1,000行を超えなければ再検討する。

残課題: シーンレベルの day メタデータ統一​

現在、各シーンファイル内の day メタデータと index.md の day_range は旧タイムライン（シーン執筆時のもの）のままである。設計文書（chapter-outline.md）の意図するタイムラインとは乖離がある。シーンレベルの day 値を統一的に再マッピングする作業は別途実施する。

新規チャプター作成の判断基準​

以下の 3条件のうち2つ以上 を満たすとき、新しいチャプターを切る。

  1. 感情の転換: 前のチャプターとは異なる感情課題が始まる
  2. 時間の飛躍: 作中時間で2日以上の空白がある
  3. 視点・構造の変化: 主要な登場人物の組み合わせが変わる、または物語構造（日常→危機等）が切り替わる

逆に、1つしか満たさない場合はシーン追加で対応し、チャプターは増やさない。

index.md の必須フィールド​

再編後の index.md には以下を含める。

id: chapter_XX
title: 第X章 - サブタイトル
