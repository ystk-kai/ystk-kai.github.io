シナリオ表記ガイド

目的​

scenarios/ の地の文と台詞は、読みやすさと実装上の解釈が両立するように表記を揃える。 特に 「」 と 『』 の使い分けは、意味と見え方を固定する。

基本ルール​

  • キャラ名「セリフ」
    • 発話として表示する台詞に使う
  • 地の文の 「」
    • 口に出された言葉の引用
    • 頭の中で反復している言葉
    • 一語だけを強調して意味を立てる時
  • 地の文の 『』
    • 物に書かれている文字
    • 掲示、プレート、張り紙、書類見出し、印刷物の題名
    • 画面上に見えている送信者名や件名

記号ルール​

  • 三点リーダは …… を使う
  • ダッシュは ―― を使う
  • 地の文は、意図的な言い切りや中断を除いて 。 で閉じる
  • 感嘆や疑問は、台詞では ？ ！ を使ってよいが、地の文では多用しない
  • 一段落は、意味の切れ目が読める長さで止める。短く切るのは強調点だけに使う

地の文と内心の使い分け​

地の文（ナレーション）と内心（主人公の心の声）は構文で区別する。

  • > — 地の文（三人称寄りのナレーション、客観描写）
  • >> — 内心テキスト（主人公の一人称の心の声、内省）
> 放課後の廊下を歩く。
>> ……あの声が聞こえた気がした。
> 足を止めた。
  • >> で書かれたテキストは、画面上で 半透明（opacity: 0.72）で表示され、地の文と視覚的に区別される
  • >> のみの行は段落区切り（空行扱い）として使える
  • 既存の > のみのナレーションはこれまでと同じ表示のまま（後方互換）

使い分けの基準​

構文用途例
>状況描写、動作の報告> 廊下の先にある扉が、少しだけ開いていた。
>>視点人物の感想、感情、回想>> ……さっきの声が、まだ耳の奥に残っている。
>客観的な観察> 窓の外はもう暗い。
>>主観的な解釈>> 暗いのは、外だけじゃない。
  • 迷ったら > を使う。>> は明確に「心の声」と判断できる場面に限定する
  • 地の文が長く続く場合に >> を挟むと、視覚リズムの変化が生まれる

インラインポーズ​

テキスト中に {p:N} を挿入すると、文字送りアニメーション中にその位置で N ミリ秒の間（ま）を入れる。

> ……あの声が{p:500}聞こえた気がした。

$saya「あの日のこと{p:300}覚えてる？」
  • N はミリ秒単位で指定する（{p:500} = 0.5秒）
  • スキップモード時はポーズを無視して即時表示する
  • 画面上のテキストには {p:N} は表示されない
  • ルビ構文 {親文字|ルビ} とは | の有無で区別され、競合しない
  • 使いすぎると冗長になるため、沈黙・ためらい・余韻など効果的な場面に限定する

ルビルール​

  • ルビは {親文字|ルビ} を使う
  • キャラクター名のルビは常に振る。地の文・台詞内を問わず、名前が出るたびに必ずルビをつける
    • 姓: {綾瀬川|あやせがわ}, {四辻|よつつじ}, {穂坂|ほさか}, {七瀬|ななせ}, {御子柴|みこしば}, {白瀬|しらせ}, {久瀬|くぜ}, {水城|みずき}
    • 名: {咲夜|さや}, {千紘|ちひろ}, {結月|ゆづき}, {湊|みなと}, {朔|さく}, {凪|なぎ}, {真帆|まほ}
    • フルネーム: {穂坂結月|ほさか ゆづき} のように一括で振ってもよい
    • ただし @chara, @hide, @name 等のコマンド引数や、セリフの話者名（行頭の キャラ名「」 の部分）には振らない
  • 中学生が読めない漢字・難読語にはルビを振る（例: {反芻|はんすう}, {痕跡|こんせき}, {譜面台|ふめんだい}）
  • 当て字や意図的な読み替えにもルビを使う（例: {運命|さだめ}）
  • 読みを伏せたい段階では振らない
  • 物に書かれた文字へルビを足す場合も、紙面そのものに振ってあるように見せたいのか、視点人物が頭の中で読み下しているのかを分ける
  • 雰囲気づくりだけの過剰なルビは避ける

年代表現ルール​

  • 生活感は現代日本相当の2020年代半ばに揃える
  • ただし西暦、元号、実在SNS名、実在アプリ名、流行語は安易に出さない
  • 端末は スマートフォン、連絡手段は メッセージアプリ や グループチャット のような一般名で書く
  • 背景と矛盾する時代語彙は避ける

会話ルール​

  • 高校生同士の会話では、説明のための不自然な名乗りを避ける
  • 初対面でも、同じ学校内なら 苗字 + クラス 程度を基本にする
  • フルネームが必要な場面は、改まった自己紹介、書類、第三者への説明に限る
  • 地の文で情報を補うために、台詞を不自然に硬くしない

名前欄ルール​

  • 名前欄は、原則として その時点で主人公が認識している最短の自然な表示 を使う
  • 未認識の人物は 女子生徒 先輩 一年生 など、視点に沿った仮ラベルで表示する
  • 初回の名前判明直後は、必要なら一時的にフルネームを使ってよい
  • 通常の会話運用では、名前欄はフルネーム固定にしない
  • 本文の会話温度や関係性とズレないよう、名前欄は 咲夜 千紘 結月 のような短い表示を優先する
  • 先輩後輩の距離を名前欄で演出したい場合でも、常時 四辻千紘 のようなフルネームは避ける
  • フルネーム固定を使うのは、キャラ一覧、ルート名、資料見出し、セーブデータ補助表示など本文外のUIを優先する

例:

  • 未認識: 女子生徒
  • 姓まで判明: 綾瀬川
  • 読みが開いた後: 咲夜

例:

  • スマートフォンが震えた
  • グループチャットに写真が貼られた
  • 配信アプリのDM のような、時代を狭く固定する固有寄り表現は避ける

使い分け​

「」 を使う場面​

  • 誰かの発言を地の文で受ける時
  • 主人公が言葉として思い返している時
  • 一語だけを切り出して、感情や意味の焦点を置く時

例:

  • 「大丈夫？」と聞かれるたびに、答える手間だけが増えた。
  • その「同じだね」が、妙に残った。

『』 を使う場面​

  • 目の前の掲示や書類に印字された文字を読む時
  • プレートやラベルの文字を、そのまま見せる時
  • スマートフォンや紙面で、表示名そのものを見ている時

例:

  • 『旧音楽室』と書かれたプレートが、夕暮れに半分だけ読めた。
  • 教員向けの書類には『参加者の体調不良により継続が困難』と書かれていた。
  • 送信者の名前は『久瀬真帆』だった。

避けること​

  • 物に書かれた文字を、理由なく地の文の 「」 で括る
  • 強調でも引用でもない単語を、癖で 「」 に入れる
  • 『』 を感情強調に使う
  • …… と ... を混在させる
  • ―― と —— を混在させる
  • 地の文を、理由なく体言止めや句点なしで連発する

実装上の注意​

  • シナリオコンパイラが台詞として解釈するのは、行頭の キャラ名「セリフ」 形式だけ
  • 地の文内の 「」 と 『』 は、そのまま文字として扱われる
  • ルビは本文にも台詞にも使えるが、Markdown のバッククォートで囲わない
  • そのため、地の文では意味に応じて 「」 と 『』 を使い分けてよい
  • >> は内心テキストとしてコンパイルされ、isInnerThought: true フラグが付く。> との判定順序はパーサー内で >> が先
  • {p:N} はコンパイラを素通りし、フロントエンドの rubyParser で解釈される。ルビの {親文字|ルビ} は | を必須とするため競合しない

セリフの品質基準​

主人公（朔）の応答密度ガイドライン​

場面応答の長さ例
シリアス・傾聴短い。沈黙も武器「……待つ」「言いにくいなら、そのままでいい」
日常会話普通。一文で完結「手伝えるところだけ手伝うよ」
掛け合い・コメディやや長い。ツッコミとして着地させる「善意の窃盗という概念はない」

避けるべきセリフパターン（全キャラ共通）​

  1. 単語応答の連続: 3文字以下の応答が3回以上連続すると、壁打ちになる
  2. オウム返し（ECHO）: 相手の台詞をそのまま疑問形にして返すだけの応答。情報が増えない
  3. ツッコミ不在: 掛け合い・コメディ場面で、ボケに対する着地がない
  4. 壁打ち（WALL_BOUNCE）: 掛け合い場面で応答が短すぎて、相手が一方的に話している状態

ツッコミの基本型​

受けた内容を具体的に引用して返す。「ひどいな」で止めず、何がひどいか一言足す。
