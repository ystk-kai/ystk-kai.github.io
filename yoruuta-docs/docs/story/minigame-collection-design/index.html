ミニゲーム収集設計

目的​

本資料は、物語中に点在するミニゲームを 収集対象 として扱い、 タイトル画面から再挑戦できる恒久コンテンツへ落とし込むための設計書である。

狙いは次の三つにある。

  • ストーリー分岐を 探す楽しさ に変換する
  • 補助キャラクターや場所に、会話以外の接点を持たせる
  • 本編を必須化せず、やり込み層として別の満足を作る
  • 本編に疲れたプレイヤーにも、作品へ戻る休憩所を作る

基本方針​

1. 本編の主導権を奪わない​

  • ミニゲームはメインルート解放条件に使わない
  • 上手さで正規エンドの価値を上下させない
  • 本編で楽しめなかったプレイヤーにも、別軸の滞在理由を与える

2. 発見そのものを報酬にする​

  • どこで遊べるかを明示しすぎない
  • サブシーンや寄り道の中に 遊べる場所 を埋める
  • 一度見つけたミニゲームは、以後タイトルからいつでも選べる

3. 物語文脈を持つゲームだけを採用する​

  • ただ遊べるだけのアーケード集にはしない
  • その場所、その相手、その場面だから成立する遊びを選ぶ
  • 1プレイ 2〜4 分程度で終わる密度を基準にする

4. 収集と滞在を分ける​

  • 発見した 時点でミニゲームは解放される
  • 遊んだ 時点で少しずつメダルが蓄積する
  • 上手いほど得をするが、下手でも前進感は残す
  • 収集だけでも楽しく、継続プレイにはメダルという別の動機を用意する

導入意図​

本作の本筋は、傷の扱い方と距離感の物語である。 そのため、ミニゲームは主菜ではなく、日常の温度を補強する副菜として扱うべきである。

一方で、現在の構造は 選択と会話 に重心が強く、探索的な遊びは薄い。 そこで、物語の節目に小さなゲーム体験を点在させると、 プレイヤーは どの寄り道に何があるのか を探し始める。

この感覚は、単なる周回ではなく 宝探し に近い。 特に湊のような、日常と寄り道をつなぐ役には非常に相性が良い。

同時に、この層には 本編の感情負荷から一度離れる場所 という役割も持たせる。 重いシーンを読み進めるのがしんどい時でも、タイトルから短く遊べる場所が残っていれば、 プレイヤーは作品そのものを閉じ切らずに済む。

システムの骨子​

解放フロー​

  1. ストーリー中の対象シーンに入る
  2. そのシーンでミニゲームの存在を発見する
  3. 発見時点で、そのミニゲームがグローバルで解放される
  4. タイトル画面の ミニゲーム から再挑戦可能になる

メダルフロー​

  • 各ゲームで メダル を獲得できる
  • card_21 と lucky_reels は所持メダルを賭けてから遊ぶ
  • 賭け枚数を上げるほど、払い戻しも大きくなる
  • 負けると所持メダルは減るが、累計獲得メダルは減らない
  • メダルは 現在所持 と 累計獲得 を分けて管理する
  • 隠しENDや追加報酬は 累計獲得メダル を参照する

隠しENDフロー​

  • 全対象ミニゲームを解放済み
  • 累計獲得メダルが目標値に到達

上記を満たした時だけ、ミニゲーム一覧 の末尾に 隠しEND用のストーリー項目を追加する。 本編タイトル導線からは分離し、任意のやり込み報酬として扱う。

  • 再生時の見た目は通常ADVと完全に揃え、開始は checker_out、終了は checker_in を使う

メダル設計の原則​

1. 入口は残し、賭け台はちゃんと痛い​

  • arcade_stack は無料で遊べる主力台として残す
  • card_21 と lucky_reels は賭け台として、負ければ所持メダルが減る
  • ただし賭け額は小刻みにし、初期の試行錯誤で詰みにくくする
  • 本編に疲れて逃げてきたプレイヤーを閉め出さないため、無料台を必ず一本残す

2. うまさの報酬も残す​

  • 高スコアや連勝は、より大きい賭けを安全に乗せられる形で返す
  • ベストスコアは副次目標として残す
  • ただし隠しEND条件はベスト値に直結させない

3. ゲームごとに欲求の種類を分ける​

  • 技術で伸びるゲーム
  • 判断で勝つゲーム
  • 変動報酬を気軽に回すゲーム

この三つを揃えると、気分に応じて戻る理由が変わる。

初期3本の確定案​

最初に実装するのは、入力負荷が軽く、モバイルでも破綻しにくく、 かつメインヒロインの感情線に直接触れすぎない次の3本とする。

arcade_stack放課後アーケード左右スワイプ + 回転タップchapter_01 / scene_sub_minato_01技術で伸びる主力台。休憩所の顔になるプレイ結果に応じてメダル増減

採用理由​

  • arcade_stack

    • 湊の寄り道導線と相性が最も良い
    • ゲームセンターという場所が ミニゲーム群の入口 として機能する
    • ちゃんとした落ち物パズルとして、長く遊べる軸にしやすい

  • card_21

    • 1ラウンドが短く、休憩所に向く
    • 反射神経ではなく判断で勝てる
    • いろはの静かな観察と相性が良い

  • lucky_reels

    • もっとも軽い操作で回せる
    • 変動報酬が強く、もう一回 を作りやすい
    • あかりの派手さやネオンとの相性が良い

配置詳細​

1. arcade_stack​

  • 章: chapter_01
  • シーン: scene_sub_minato_01.md
  • 導線:
    • 湊サブへ入る
    • 自販機前会話の後半で、湊が まだ帰るには早い と寄り道を提案
    • ゲーセンの1プレイだけ付き合わされる
  • 分岐位置:
    • chapter_01 のサブノードから枝分かれする任意導線
    • 本編には直接影響しないが、初回解放でタイトルのミニゲーム一覧が開き始める

2. card_21​

  • 章: chapter_02
  • シーン: scene_02b_minato_iroha_banter.md
  • 導線:
    • 湊といろはの軽口シーンに入る
    • 帰り際、駅前のゲームコーナーにある静かなカード台の話題が出る
    • いろはが こういうのは確率を読めば少しだけ楽です と言う
  • 分岐位置:
    • chapter_02 のいろは側サブノードから枝分かれする任意導線
    • うるさすぎない台として、タイトル休憩所の中間ポジションを担う

3. lucky_reels​

  • 章: chapter_02
  • シーン: scene_05_preparation.md
  • 導線:
    • あかりの掲示・導線づくりを見る
    • 派手さと視線誘導の話から、駅前のネオン台の話題に繋ぐ
    • タイトルの休憩所では、もっとも気軽に回せる台として置く
  • 分岐位置:
    • chapter_02 本編ノードから派生する軽い導線
    • 本編の感情線ではなく、また戻りたくなる入口 として扱う

優先実装順​

Phase 1​

  • arcade_stack
  • card_21
  • lucky_reels

理由:

  • 遊びの軸が分散している
  • 技術・判断・運の三軸が揃う
  • 本編に疲れた時の逃げ込み先として性格が分かれる
  • UI と入力設計の共通部を作りやすい

Phase 2​

  • broadcast_cue
  • phrase_trace
  • poster_route

理由:

  • 物語文脈は強いが、演出依存がやや高い
  • 先に共通のミニゲーム基盤を固めた方が安全

湊ゲーセン案の位置づけ​

arcade_stack は、単独のサブEND装置ではなく、 ミニゲーム収集レイヤーの 入口 として置くのが最も強い。

理由は以下の通り。

  • 湊は寄り道へ連れ出す役として自然
  • ゲーセンは 遊びが置いてある場所 として説明不要
  • タイトル画面のミニゲーム選択と、世界観上の接続が取りやすい

この時、重要なのは テトリスを実装したから面白い ではなく、 湊と遊んだ記憶がコレクションの最初の一個になる ことにある。

タイトル画面との関係​

タイトル画面には、既存の はじめる / つづきから / 設定 に加えて ミニゲーム を追加する。

  • 未解放時は非表示、または ???
  • 1本でも解放すると選択可能
  • ミニゲーム一覧画面で、現在メダル・累計メダル・各台の記録を確認可能

ここで大事なのは、タイトル側では 物語の途中の失敗 を引きずらないことだ。 タイトルから遊ぶ時は、純粋な再挑戦として扱う。

シナリオ分岐マップ表示​

  • Docsaurus の シナリオ分岐マップ には、各章のミニゲーム解放ポイントを専用ノードで表示する
  • arcade_stack は chapter_01_sub_湊 から派生
  • card_21 は chapter_02 のいろは側サブ導線から派生
  • lucky_reels は chapter_02 のあかり導線から派生
  • マップ上では 任意 / 解放で一覧追加 と分かるラベルにする

隠しENDの方針​

役割​

  • 本編の理解に必須ではない
  • 収集とやり込みの総決算として機能する
  • 主ヒロインのエンディングより上位に見せない
  • メダル 1000 枚の消費で好きなストーリーを選んで解放する購入制

内容の温度​

  • 本編級の長編にはしないが、単発の一発ネタでは終わらせない
  • 各話 2 シーン以上で、掛け合いとオチが育つ長さを確保する
  • ギャグ寄りで、遊び切った後の空気抜きとして機能させる
  • ただし本編の印象を崩さず、各キャラの魅力が強まる方向に限定する
  • ご褒美は恋愛成就より このキャラをもっと好きになる一面 に置く

推奨案​

  • 配置: ミニゲーム一覧 の最下段でのみ解放
  • 内容:
    • 第1話は湊を主役にした 軽口と保護者役 のギャグ寄りサブストーリー
    • 第2話は御子柴いろはを主役にした 静かな観察と刺さる一言 のサブストーリー
    • 第3話はあかりを主役にした 派手さと面倒見の良さ を同時に見せる締めのサブストーリー
  • 着地: 3本とも 遊び場だから少しだけ前に出る 温度に止め、本編の感情クライマックスは食わない

本編分岐との関係​

  • メインルート条件には使わない
  • BAD END 回避条件にも使わない
  • サブシーン内での台詞差分、CG差分、短い後日談の条件には使ってよい

これにより、ミニゲームが 遊ばないと損 ではなく 遊ぶと別の温度が見える 位置に収まる。

運用上の注意​

  • 1章内に複数本を詰め込みすぎない
  • 傷の直後に反射ゲームを置かない
  • 賭け台は負けで所持メダルが減るが、無料台が進行の逃げ道として残るようにする
  • タイトルからの再挑戦で、シナリオ進行が壊れないよう完全分離する

ペルソナ評価条件​

  • ミニゲームENDは、既存の検証ペルソナで ギャグ・日常パート と キャラクター魅力 の両方を落とさない
  • 笑える だけでは不可。A/C/D から 本編の空気を壊していない 評価を取ること
  • B/E/F/G からは 短くて分かりやすく、キャラを好きになれる 評価を取ること
  • 目標は全ペルソナで総合 7.5 以上、コアターゲットの A/C/E は 8.5 以上
